## 🚀 核心功能

### 🤖 自動化資料處理
串接 [FinMind](https://github.com/FinMind/FinMind) API,自動抓取台股過去 5 年的現金流量與稅前淨利。

### 🧠 智慧年化邏輯
解決台灣財報「累計型數據」的判讀痛點:
Q2 報表寫的不是 Q2 單季數字,而是 Q1+Q2 的累計值。
本系統自動拆解為單季增量,並嚴格檢查資料完整性(缺 Q1 整年跳過,避免錯誤計算)。

### 📐 六大現金流檢驗法則
| 編號 | 檢驗項目 | 白話翻譯 |
|------|---------|---------|
| 1 | 營運現金流年增率 | 公司核心造血能力是否變強? |
| 2 | 投資現金流紀律 | 有沒有過度擴張? |
| 3 | 投資效益評估 | 燒掉的錢有沒有換來營運成長? |
| 4 | 融資活動合理性 | 有沒有「借錢去投資」的危險訊號? |
| 5 | 現金流品質(A > D) | 賺到的是真金白銀還是帳面數字? |
| 6 | 現金餘額穩定度 | 口袋夠不夠深? |

## 🏗️ 技術架構

```
使用者輸入股票代號
        ↓
   前端介面(HTML)
        ↓
  n8n 自動化工作流
        ↓
本專案:Flask API(資料處理 + 六大法則評分)
        ↓
   FinMind API(資料源)
        ↓
  GPT-5 Mini(AI 撰寫分析報告)
        ↓
   Google Drive(自動存檔)
```

| 層級 | 技術選型 |
|------|---------|
| 後端 API | Python + Flask |
| 資料處理 | Pandas(時間序列、年化轉換) |
| 部署 | GitHub + Zeabur(CI/CD 自動化) |
| 跨域支援 | Flask-CORS |
| 工作流串接 | n8n + OpenAI API + Google Drive API |

最初我想一次寫完整套系統,結果一個 bug 卡三天。

這個思維,反過來也改善了我的教學設計。

### 3. 領域知識是最大的優勢

## 🛠️ 在你自己的電腦上跑跑看

**前置需求**:Python 3.9 以上

```bash
# 1. Clone 專案到你自己的電腦
git clone https://github.com/mjib007/taiwan-stock-cashflow-api.git
cd taiwan-stock-cashflow-api

# 2. 安裝套件
pip install -r requirements.txt

# 3. 啟動服務(會跑在你電腦的 port 5000)
python app.py
```

啟動後,在瀏覽器打開以下網址測試:

- `http://localhost:5000/` — API 首頁
- `http://localhost:5000/analyze/2330` — 完整六大法則分析
- `http://localhost:5000/raw-data/2330` — 取得單季原始資料
- `http://localhost:5000/data-range/2330` — 檢查資料完整性

> 💡 **說明**:本專案為**個人教學用途**,線上服務僅供作者本人課程使用。
> 若你想體驗,歡迎 clone 到自己電腦執行——資料源 FinMind API 為公開免費服務,無需金鑰。

## 📖 API 輸出說明

回傳結果都是 JSON 格式。看起來像一堆英文,別怕,以下解說各個端點的重點欄位。

### 🔍 `/analyze/2330` — 完整六大法則分析

這個端點會吐出**評分結果**,主要欄位:

| 欄位 | 說明 |
|------|------|
| `rules_analysis` | 六大法則各自的評分(GOOD / AVERAGE / POOR) |
| `summary` | 整體評分(滿分 18 分)與投資建議文字 |
| `data_points` | 分析使用了幾個季度的資料 |
| `latest_date` | 最新一筆財報日期 |

**評分對照**:

| 狀態 | 分數 | 顏色 |
|------|------|------|
| GOOD | 3 分 | 🟢 |
| AVERAGE | 2 分 | 🟡 |
| POOR | 1 分 | 🔴 |

**投資建議分級**:

- 總分 ≥ 80% → 現金流狀況優秀,值得關注
- 總分 60%~80% → 現金流狀況一般,需持續觀察
- 總分 < 60% → 現金流狀況不佳,需謹慎評估

### 💰 `/raw-data/2330` — 取得單季原始資料

這個端點會吐出**每一季的現金流原始數字**(已從累計值轉換為單季):

| 欄位代號 | 中文意義 | 正常情況 |
|---------|---------|---------|
| A | 營運現金流 | 正值越大越好 |
| B | 投資現金流 | **負值代表公司在投資設備,正常** |
| C | 融資現金流 | 負值代表還債或發股利 |
| D | 稅前淨利 | 公司賺的帳面利潤 |
| E | 期末現金餘額 | 公司口袋裡的錢 |

> 📝 **單位說明**:數字單位為**新台幣元**。除以 100,000,000 即為「億元」。
> 例如 `350000000000` 就是 **3,500 億元**。

### 📋 `/data-range/2330` — 檢查資料完整性

回傳這檔股票有哪些季度的資料、哪些季度因為缺漏被自動跳過。
**用途**:做完整分析前,先確認資料是否充足、有沒有缺洞。

## ❓ 常見問題(FAQ)

### Q1:`pip install` 時 pandas 安裝失敗,出現 `Could not find vswhere.exe`?

這是我自己 clone 自己專案到本地測試時踩到的坑(笑)。

**原因**:Windows 環境下,如果 `requirements.txt` 鎖死特定版本(例如 `pandas==2.1.1`),
而該版本沒有預先編譯好的 wheel 檔,pip 會嘗試從原始碼編譯,需要 Visual Studio C++ 工具。

**解法**:本專案的 `requirements.txt` 已從鎖死版本改為**寬鬆版本標記**(如 `pandas>=2.1.0`),
讓 pip 自動找最新可用的預編譯版本,不需要本機編譯。

如果你 clone 下來後還是遇到此問題,請確認你下載的是最新版的 `requirements.txt`。

### Q2:啟動時出現 `ModuleNotFoundError: No module named 'flask_cors'`?

代表你跳過了「安裝套件」這一步。請先執行:
```
pip install -r requirements.txt
```
裝完再跑 `python app.py`。

### Q3:啟動時出現 `Address already in use`?

代表 port 5000 已被其他程式占用。
**解法**:關掉占用 port 5000 的程式,或修改 `app.py` 最後一行的 port 號碼(例如改成 5001)。

### Q4:呼叫 API 出現 `Unable to fetch cash flow data`?

可能原因:

- 股票代號錯誤(例如打錯數字)
- 該股票太新,FinMind 還沒收錄資料
- FinMind API 暫時不穩(稍後重試即可)
